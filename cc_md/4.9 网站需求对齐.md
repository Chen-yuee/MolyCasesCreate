
# moly产品背景
打造一个为老板服务的秘书，老板上传多种个人信息，moly 能执行日程安排、联系人记录、待办清单、会议记录、对话建议等帮助。
现在产品急需 moly memory 架构

# moly memory 架构功能

用户询问一个 query，例如“moly，我明天下午有一个 15 点的腾讯会议，关于 agent 的，请通知有关毕昇项目的成员，你帮我安排下会议“，
架构应该能根据以往上下文进行辅助回答。

# moly memory 测试方法

将 query 应摄取到的有关上下文拿出来，测试架构的输出拿出来。
比如 evidence “有个毕昇负责人 qr，在联系人库中“、“25年 4 月 1 号下午 15 点两个小时腾讯会议“
我设置一个判别模型，判别架构输出有没有用到这些 evidence 作为测试样例。比如如果测试模型没有输出毕昇负责人 qr，那就说明不对。
这个具体的测试细节，我们产品构建过程中还没到这一步。

# moly业务数据集构建

有一些开源数据集本身就有合适的长对话上下文。只是没有我们 moly 业务上query 合适的 evidence。
